# Glean for Cursor

Connect Cursor to Glean's remote MCP server and give Cursor's AI access to your company's docs, decisions, code, people, and tickets — all respecting your existing access controls.

## What you can do

Once connected, Cursor's AI can draw on your organization's actual knowledge:

- *"What's our internal API rate limit policy?"* — pulls from internal docs
- *"Who should I talk to about the data pipeline?"* — finds the right person
- *"How did we solve this problem last year?"* — surfaces past decisions and docs
- *"Show me examples of how we handle auth in our services"* — searches internal code context

## Prerequisites

- A Glean instance with MCP enabled
- A Glean API token (see [Authentication](/get-started/authentication))
- Cursor installed

## Installation

Use the [MCP Configurator](/guides/mcp) to generate your server URL and credentials. Select **Cursor** as your host, enter your Glean instance details, then follow the steps to add the server in Cursor's settings.

## Verifying the connection

After adding the MCP server in Cursor, open a new chat and ask something about your internal knowledge — for example, a question about a doc, a policy, or a person on your team. Cursor will call Glean's MCP tools automatically when it needs enterprise context.

## Available tools

The Glean MCP server exposes the following tools to Cursor:

| Tool | Description |
|---|---|
| `glean_search` | Full-text search across all connected Glean sources |
| `glean_chat` | Conversational Q&A grounded in your company's knowledge |
| `glean_get_document` | Retrieve a specific document by URL |
| `glean_list_datasources` | List available data sources |

## Configuration options

The remote MCP server is managed by Glean — no local process to maintain. Access controls are enforced per-user based on your Glean permissions.

To update your token or instance settings, use the configurator above to regenerate your server URL and replace it in Cursor's MCP settings.
