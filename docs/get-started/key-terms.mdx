# Key Terms and Concepts

Understanding these key terms will help you navigate Glean's developer platform and APIs effectively.

## Authentication & Access

| Term | Definition | Details |
|------|------------|---------|
| **API Token** | A credential used to authenticate API requests | Glean provides two distinct types:<br/>• **Client API Tokens**: For user-facing APIs (search, chat, etc.)<br/>• **Indexing API Tokens**: For content upload and datasource management |
| **User-Scoped Token** | A Client API token tied to a specific user's identity | Requests return data accessible to that user only |
| **Global Token** | A Client API token that can impersonate any user | Uses `X-Glean-ActAs` header. Only Super Admins can create these |
| **OAuth Access Token** | A token issued by your identity provider | From Azure, Google, Okta, etc. for Client API authentication |
| **Scopes** | Permissions defining API endpoint access | Examples: `CHAT`, `SEARCH`, `COLLECTIONS` |

## APIs & Endpoints

| Term | Definition | Use Case |
|------|------------|----------|
| **Client API** | User-facing API for search, chat, and AI interactions | End-user integrations and applications |
| **Indexing API** | Administrative API for content upload and datasource management | Content ingestion and administrative operations |
| **REST API** | HTTP-based API using standard methods and JSON | GET, POST, PUT, DELETE operations with JSON payloads |

---

## Data & Content

| Term | Definition | Examples |
|------|------------|-----------|
| **Datasource** | A configured content source in Glean | Confluence, Slack, custom integrations |
| **Document** | A piece of content indexed in Glean | Files, web pages, database records, structured content |
| **Index** | Glean's searchable content repository | Content must be indexed before search/AI can use it |
| **Permissions** | Access controls determining content visibility | Glean enforces same permissions as source systems |
| **Bulk Indexing** | Process for uploading large document volumes efficiently | Uses batch operations for performance |

---

## AI & Search

| Term | Definition | Capabilities |
|------|------------|--------------|
| **Chat** | Conversational AI interface | Answer questions, summarize content, provide insights |
| **Search** | Enterprise search functionality | Finds relevant content across all connected datasources |
| **Agent** | AI-powered assistant | Customizable for specific tasks or domain questions |
| **Summarization** | AI-powered content summarization | Creates concise summaries of documents or search results |
| **Citations** | References to source documents | Provides transparency and verification for AI responses |

## Platform Features

| Term | Definition | Purpose |
|------|------------|---------|
| **Collection** | Curated group of documents or resources | Organized around specific topics or purposes |
| **Answer** | Structured Q&A entry | Provides authoritative responses to common questions |
| **Announcement** | Company-wide communications | Updates distributed through Glean |
| **Shortcut (GoLinks)** | Short, memorable links | Redirect to frequently accessed resources |
| **Pin** | Featured content | Appears prominently in search results or contexts |

## Development & Integration

| Term | Definition | Available Languages/Details |
|------|------------|----------------------------|
| **SDK (Software Development Kit)** | Pre-built libraries that simplify API integration | Python, TypeScript, Go, Java |
| **Web SDK** | JavaScript library for web applications | Embeds Glean's search and chat functionality |
| **MCP (Model Context Protocol)** | Protocol for AI model integration | Allows secure access to Glean's knowledge and search |
| **Webhook** | HTTP callbacks for event notifications | Notifies applications when specific events occur |
| **Rate Limiting** | API request restrictions | Ensures system stability within time periods |

---

## User Roles & Permissions

| Role | Permission Level | Key Capabilities |
|------|------------------|------------------|
| **Super Admin** | Highest level | Create global tokens, manage system-wide settings, access all features |
| **Admin** | Administrative level | User token creation, datasource configuration, most management features |
| **API Token Creator** | Limited administrative | Create API tokens for own use only |
| **End User** | Standard user | Access via web interface or integrated applications |

## Technical Concepts

| Term | Definition | Example |
|------|------------|---------|
| **Instance** | Your organization's unique Glean environment | `yourcompany-be.glean.com` |
| **Bearer Token** | Authentication method using Authorization header | `Authorization: Bearer <token>` |
| **HTTPS/SSL** | Secure HTTP protocol | Required for all API communications |
| **JSON** | JavaScript Object Notation data format | Used for API requests and responses |
| **MIME Type** | File type classification | `application/pdf`, `text/html` |

## Integration Patterns

| Pattern | Definition | Use Cases |
|---------|------------|-----------|
| **Streaming** | Real-time data delivery method | Chat responses, live updates |
| **Pagination** | Retrieve large datasets in manageable chunks | Large result sets, performance optimization |
| **Filtering** | Restrict results based on specific criteria | Content type, date, source filtering |
| **Faceted Search** | Multi-category filter refinement | Simultaneous filtering across multiple categories |

## Common Acronyms

| Acronym | Full Term | Definition |
|---------|-----------|------------|
| **API** | Application Programming Interface | Set of protocols for building software applications |
| **SDK** | Software Development Kit | Collection of tools for developing applications |
| **REST** | Representational State Transfer | Architectural style for web services |
| **HTTP** | Hypertext Transfer Protocol | Protocol for transferring data over the web |
| **JSON** | JavaScript Object Notation | Data interchange format |
| **CRUD** | Create, Read, Update, Delete | Basic operations for data management |
| **JWT** | JSON Web Token | Compact token format for securely transmitting information |
| **SSO** | Single Sign-On | Authentication system allowing access to multiple applications |
| **SAML** | Security Assertion Markup Language | Standard for exchanging authentication data |
| **OIDC** | OpenID Connect | Identity layer on top of OAuth 2.0 |
