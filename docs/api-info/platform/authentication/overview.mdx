# Platform API Authentication

Requests to the Platform API are authenticated with a Glean API token passed as a
bearer token in the `Authorization` header.

```bash
curl https://{instance}-be.glean.com/api/search \
  -H "Authorization: Bearer <API_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{ "query": "quarterly planning" }'
```

Replace `{instance}` with your Glean instance name (typically your email domain
without the TLD) and `<API_TOKEN>` with a valid token.

:::info Experimental
The Platform API is experimental. Authentication scopes and token requirements may
change as the API evolves toward general availability.
:::

## Getting a token

Glean API tokens are issued from the Glean admin console. For a deeper explanation
of the available token types and how permissions are enforced, see the
[Client API authentication guide](../../client/authentication/overview), which
describes the same token model used across Glean's REST APIs.
